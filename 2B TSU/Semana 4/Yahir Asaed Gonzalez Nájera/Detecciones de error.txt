Errores de Compilación (Sintaxis)
Línea 23: while (salir = false) — Uso de asignación en lugar de comparación ==.

Línea 49: if (nota >= 0 && nota <= 10); — El punto y coma final anula la condición del if.

Línea 53: }else — Error de vinculación; el else queda huérfano por el ; de la línea 49.

Línea 50-51: Falta de llaves { } para agrupar las instrucciones de suma y conteo dentro del if.

Línea 70: if (promedio >= 9) — Falta de llave de apertura { para el bloque de código.

Línea 74: ; innecesario después del else if que rompe la secuencia de la condición.

Línea 89: while (i < cantidad); — El punto y coma genera un bucle infinito que no ejecuta el bloque siguiente.

Línea 105: Falta de break; al final del Case 4, provocando que se ejecute el Case 5 por error.

Errores Lógicos

Línea 44: i <= n en el ciclo for. Causa un error de "fuera de rango" o pide una nota de más (si pides 5 notas, pide 6).

Línea 90: if (total / cantidad >= 7) dentro del while. Intenta calcular aprobados usando el promedio grupal, no la nota individual.

Línea 90: if (total / nota >= 7) — Error de variable; usa la última nota ingresada en lugar del divisor correcto.

Línea 67: total / cantidad — No valida si cantidad es 0 antes de dividir, lo que causaría un error fatal en el programa.

Línea 87: int i = 0 — El contador del while se reinicia cada vez que entras al Case 3, pero no hay una lista de datos reales que recorrer.

Línea 103: El Case 4 no reinicia las variables aprobados y reprobados, dejando datos sucios para la siguiente captura.

Línea 10: int total = 0 — Al ser entero, el promedio pierde todos los decimales en la división de la línea 67.

Malas Prácticas

Línea 44: int i = 0 — Iniciar en 0 para mostrar "Nota #0" es confuso para el usuario final.

Línea 42: int n — Nombre de variable poco descriptivo; debería ser algo como limiteNotas.

Línea 112: Falta de sc.close() — No cerrar el Scanner es una mala práctica de gestión de memoria en Java.

Línea 107: Falta de un caso default: en el Switch para manejar cuando el usuario elige un número que no es del 1 al 5.

Línea 68: Repetición de la operación total / cantidad. Es mejor guardar el resultado en una variable una sola vez para ahorrar recursos.
