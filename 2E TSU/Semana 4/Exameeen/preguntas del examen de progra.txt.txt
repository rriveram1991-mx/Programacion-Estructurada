1. ¿Por qué el sistema entra en un comportamiento inesperado desde el inicio?
Porque el punto y coma en la línea while (opcion != 3); hace que el bucle no tenga cuerpo.

2. ¿Por qué el sistema acepta ventas negativas?
Porque la condición if (venta < 0); no tiene cuerpo, por lo que se ejecuta siempre el bloque de código siguiente.

3. ¿Por qué el total vendido no se acumula correctamente?
Porque la asignación totalVentas = venta; reemplaza el valor anterior de totalVentas con el valor de venta.

4. ¿Qué problema lógico existe en el for?
El bucle for se ejecuta una vez más de lo necesario, lo que puede provocar un error si contadorVentas es 0.

5. ¿El sistema realmente termina cuando se elige la opción 3? Justifica tu respuesta.
No, porque el bucle while sigue ejecutándose después de elegir la opción 3. La condición opcion != 3 se evalúa como falsa, pero el bucle ya se ha iniciado.

Explicación del Sistema Actual y Deseado

- Actualmente, el sistema tiene varios errores que provocan un comportamiento inesperado.
- El sistema deseado debería:
- Mostrar un menú con opciones para registrar ventas, mostrar el total vendido y salir.
- Permitir registrar varias ventas y validar que el monto ingresado sea mayor a 0.
- Mostrar el total acumulado de ventas.
- Terminar cuando el usuario elija salir.
